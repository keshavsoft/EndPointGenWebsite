KSTable Engine Firefox Extension

Front-End Client Integration & Sibling Repositories Guide

Overview

The KSTable Engine Firefox extension is a front-end injector companion. It injects high-performance customized tables, headers, and layouts dynamically into web pages.

Rather than bundling large layout and rendering code scripts into each individual page served by the Express API, the extension registers these libraries globally inside the browser context, making them accessible to any client page served by the API.

Workspace Repositories Involved

The extension source files are compiled and built from 6 different workspace repositories. They work in unison to maintain and bundle the browser extension:

KsWebExtension
The browser extension root directory containing the packaging scripts, manifest settings, content scripts, and build outputs.
tailwind-header-dom
Source of the header rendering scripts (ksheader.js and ksheader.min.js).
ks-web-comp-table
Source of the core custom element web components (KSComponents.js).
tailwind-table-dom-comp
Source of the high-performance grid table modules (kstablecomp.js).
tailwind-table-dom-comp-show
Source of the simple view table layouts (kstableonly.js).
tailwind-vertical-dom
Source of the vertical list layout rendering system (ksvertical.js).

Development & Run Instructions

To sync changes and load the browser extension in Firefox, use the following developer pipeline:

  • Run Sibling Builds: Ensure you run the build processes (npm run build or equivalent) in the respective sibling repository directories first to generate their latest code outputs.
  • Execute Extension Sync: Open your terminal in the KsWebExtension directory and run: 
    npm run update-src
  • Load Add-on in Browser: Open Firefox, type about:debugging in the URL bar, select "This Firefox", click "Load Temporary Add-on...", and select the manifest.json file in KsWebExtension.
